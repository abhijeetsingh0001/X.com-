# X (Twitter) Clone 🐦

A responsive, static front-end clone of the **X (formerly Twitter)** home feed UI, built with plain **HTML** and **Tailwind CSS v4**. This project replicates the three-column layout of X — left navigation sidebar, main feed, and right sidebar (trends & who-to-follow) — using utility-first CSS with no JavaScript framework.

## 📖 About the Project

This project is a UI/UX clone built for learning and practicing **Tailwind CSS v4**. It recreates the core layout and visual design of the X home timeline, including:

- A fixed left-hand navigation sidebar with icons and labels
- A central feed with a "compose post" box and sample posts
- A right sidebar with trending topics and follow suggestions

It is purely a front-end/static UI project — there is no backend, authentication, or real data. All content (posts, avatars, trends) is hardcoded as placeholder/sample data.

## ✨ Features

- **Responsive layout** — Sidebar collapses to icon-only view on smaller screens (`md:` breakpoints) and expands with labels on larger screens
- **Left navigation menu** — Home, Explore, Notifications, Grok, Lists, Bookmarks, Communities, Premium, Profile, More
- **Post composer** — "What is happening" input with action icons (image, GIF, poll, emoji, calendar, location)
- **Sample post card** — Avatar, username, handle, timestamp, image attachment, and engagement icons (reply, repost, like, views)
- **"For You" / "Following" tabs** with an active tab indicator
- **Right sidebar widgets**:
  - Search bar
  - "What's Happening" trending topics list
  - "Who to Follow" suggestions
- **Google Material Symbols** used throughout for icons
- **Dark theme** matching X's default black/white color scheme

## 🛠️ Tech Stack

| Technology | Purpose |
|---|---|
| [Tailwind CSS v4](https://tailwindcss.com/) | Utility-first styling |
| [@tailwindcss/cli](https://tailwindcss.com/docs/installation) | Compiles `index.css` → `output.css` |
| HTML5 | Markup |
| [Google Material Symbols](https://fonts.google.com/icons) | Icon set (loaded via Google Fonts CDN) |

> **Note:** This project uses Tailwind CSS v4's new CSS-first configuration (`@import "tailwindcss";`) and the standalone CLI — there is no `tailwind.config.js`, PostCSS, or JS bundler involved.

## 🚀 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- npm (comes bundled with Node.js)

### Installation

1. Clone the repository
   ```bash
   git clone https://github.com/<your-username>/<your-repo-name>.git
   cd <your-repo-name>
   ```

2. Install dependencies
   ```bash
   npm install
   ```

### Running the Project

This project uses the Tailwind CLI to compile styles. To watch for changes and rebuild `output.css` automatically as you edit `index.css` or classes in `index.html`, run:

```bash
npx @tailwindcss/cli -i ./index.css -o ./output.css --watch
```

Then simply open `index.html` in your browser (or serve it with a tool like the VS Code **Live Server** extension) to view the site.

For a one-time production build (minified):

```bash
npx @tailwindcss/cli -i ./index.css -o ./output.css --minify
```

## 🎨 How the Styling Works

- `index.css` contains a single line: `@import "tailwindcss";` — this pulls in all of Tailwind's utility classes.
- The Tailwind CLI scans `index.html` (and any other project files) for class names and generates only the CSS actually used into `output.css`.
- `index.html` links directly to the compiled `output.css` file, so no runtime build step is needed once compiled.
- Responsive design is handled via Tailwind's `md:` breakpoint prefix (e.g., sidebar labels are hidden on mobile with `hidden md:block`).
